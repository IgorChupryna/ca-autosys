You can configure  CA Workload Automation AE to control the run-time behavior of each instance, including which database to connect to and how to react to error conditions. You can also set up the Notification feature to communicate problems to users in your enterprise.

You can define environment variables on CA Workload Automation AE to customize logging, network communication, and the behavior of the scheduler, application server, client utilities, and SDK.

On UNIX, you configure CA Workload Automation AE to control the run-time behavior by modifying the parameters in the configuration file and the agentparm.txt file, and by modifying the environment variables in the /etc/auto.profile file and the WAAE.txt file.

On Windows, you configure CA Workload Automation AE to control the run-time behavior by modifying the configuration parameters in either the Administrator utility or the configuration file, modifying the environment variables in either the Administrator utility or the WAAE.txt file, and modifying the agent parameters in the agentparm.txt file.

Note: On Windows, the parameters in the configuration file have corresponding fields in the Administrator utility. We recommend that you use the Administrator utility instead of the configuration file to configure CA Workload Automation AE on Windows.

Configuration File

You can configure CA Workload Automation AE by setting the parameters in the configuration file. The configuration file is specific to an instance. On startup, CA Workload Automation AE reads the configuration file to verify its behavior, including which database to connect to and how to react to certain error conditions.

On Windows, you can configure CA Workload Automation AE by either editing the fields in the Administrator utility or setting the parameters in the configuration file. The parameters in the configuration file have corresponding fields in the Administrator utility. We recommend that you use the Administrator utility instead of the configuration file to set the configuration parameters on Windows.

Important! The scheduler and the application server read the settings in the configuration file only on startup. Therefore, if you make a change that you want to implement immediately, restart the scheduler and application server or pause and resume the scheduler and application server.

The configuration file has the following name:

  • On UNIX -- $AUTOUSER/config.$AUTOSERV
  • On Windows -- %AUTOUSER\config.%AUTOSERV

AUTOSERV
Defines the name of the instance (an uppercase, three-character alphanumeric name) that is associated with the configuration file.
Default: ACE

AUTOUSER
Identifies the path to the directory that is associated with a specific instance. This directory contains the instance-wide configuration files, scheduler or application server output files, encryption files, archive output files that are generated during database maintenance, and sound files (for operating environments supporting audio functionality).

A sample configuration file is available in the $AUTOSYS/install/data/config.ACE (on UNIX) or %AUTOSYS%\install\data\config.ACE (on Windows) directory. We recommend that you make a copy of the sample configuration file before you modify it.

Notes:

  • Events are associated with a specific instance. They have a unique ID, named an eoid, which is prefixed to the instance name. This naming convention ensures the uniqueness and traceability of an event across multiple instances.
  • Before you can issue commands at the UNIX operating system prompt, the CA Workload Automation AE environment must be sourced in the shell and your UNIX user ID and password must be defined on CA Workload Automation AE. For more information about sourcing the environment and defining user IDs.

Keyword: FileSystemThreshold=(value)

Defines the minimum amount of disk space that must be available for the Scheduler log. If the amount of available disk falls below that specified by the File System Threshold setting, the Scheduler issues warning messages.
If the amount of disk space falls below 8192KB (8MB), the Scheduler issues an EP_SHUTDOWN alarm and shuts down. The Scheduler passes the File System Threshold setting to the legacy agent as part of running a job. If the amount of available disk space falls below that specified by the File System Threshold setting, the legacy agent writes dots to its log file.
If the available space drops below 8192KB (8MB), the legacy agent issues a warning and stops writing to the log file, but the service keeps running.

Valid settings are between 8192KB (8MB) and 102400KB (100MB). If the values are not within the specified range the Scheduler will issue a warning message and reset the value to the default.

Note: The FileSystemThreshold setting is valid for legacy agents only. Type 'a' agents do not support this setting.

Default: 20480 (20MB)

Keyword:EvaluateQueuedJobStarts=(value)

Specifies whether the scheduler evaluates the starting conditions of a job exiting the QUE_WAIT state due to available machine load units, the PEND_MACH state due to an online machine or the RESWAIT state due to available resources. The valid values are as follows:

0 = Off

Specifies that the scheduler immediately starts the job without evaluating the job's starting conditions.

1 = Skip same day check

Specifies that the scheduler evaluates the job's starting conditions (job dependencies, etc..) ignoring date condition checks (run_calendar, days_of_week, start_times, and start_mins) before starting the job. The scheduler considers the date conditions for the job as already met from the time the job is placed in a queued state. The scheduler starts the job as long as it is not scheduled on an excluded day or outside its run window (exclude_calendar and run_window are honored). If the starting conditions for the job are not met, the scheduler places the job in the ACTIVATED status if the job is in a running box job, otherwise it places the job in the INACTIVE status. Jobs with date conditions are rescheduled to their next start time.

2 = Include same day check

Specifies that the scheduler evaluates the job's starting conditions (job dependencies, etc..) including date condition checks (run_calendar or days_of_week, but not start_times or start_mins) before starting the job. The scheduler starts the job as long as it can run on the day it is evaluated, and is not scheduled on an excluded day or outside its run window (exclude_calendar and run_window are honored). If the starting conditions for the job are not met, the scheduler places the job in the ACTIVATED status if the job is in a running box job, otherwise it places the job in the INACTIVE status. Jobs with date conditions are rescheduled to their next start time.

Default: 1

RestartConstant
Defines the Restart Constant value used for calculating the wait time (in seconds) between attempts to restart a job.

Default: 10

RestartFactor
Defines the Restart Factor value used for calculating the wait time (in seconds) between attempts to restart a job. This value multiplies with every job restart and is used to gradually increase the number of seconds per retry attempt.

Default: 5

MaxRestartWait
Defines the maximum amount of time (in seconds) that CA Workload Automation AE waits before it tries to restart a job.

Default: 300

CA Workload Automation AE uses the following formula to calculate the amount of time (in seconds) between each attempt to restart a
job:

WaitTime = RestartConstant + ( Num_of_Trys * RestartFactor )
if ( WaitTime > MaxRestartWait ) then WaitTime = MaxRestartWait

Miscellaneous Parameters

Keyword:DateFormat=(date_format)

Specify the date format for entering and displaying dates.

Keyword:LOGROLLOVER=(value)

Specify when to roll over the Scheduler and Application Server log file.

Valid values are:

OFF = Do not roll over the log file. MIDNIGHT = Roll over the log file at midnight. SIZE(n) = Roll over the log file when it becomes larger than the value specified in number of megabytes. PURGE(n) = Remove all the application generated log files older than the value specified in number of days. MIDNIGHT,SIZE(n) = Roll over the log file at midnight or when it becomes larger than the value specified in MB.

Keyword:ISDBGACTIV=(value ,value,...)

Defines the level of trace information to return to the scheduler
and application server logs. These values can be used together by separating each keyword by a comma.

The valid values are as follows:

OFF - Returns no trace information.
OFF is the default.
MS - Add milliseconds to the time in the log output.
LIGHT - Returns light trace information.
HEAVY - Returns full trace information.
DUMP - Traces data sent and received by the product's
communication interface.
JOB - Traces the run time of a job.
COMM - Traces network communication activity at the
socket level.
GBE - Traces scheduler events as they are read from the
ujo_event table.
AFM - Traces Automation Framework Messages (AFM).
DBQUERY - Traces and measures the elapsed time of calls to the database.
EXTVJ - Returns trace data pertaining to the user verification exit.
RESOURCEn - Traces the Spectrum Automation SDK. 'n' can be a value between 1-10.

ISDBGACTIV=OFF
