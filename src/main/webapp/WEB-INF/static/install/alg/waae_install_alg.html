(UNIX) Install the Server

You can install the server using the installation wizard. The server installation sets up the database and various configuration files, and configures the server computer to run as a client also. This enables you to run jobs on a server computer.

Note: You can install the server using the CA Workload Automation AE media or by downloading the ISO file by logging in to Download Center, Products in the CA Support Online website ( http://support.ca.com).

Follow these steps:

  1. Log in as root.
  2. Mount the CA Workload Automation AE media.

  3. Change directories to the mounted file system and run the following command:

    ./wa_setup.sh

    The Welcome page appears.

  4. Click Next.
    The License Agreement page appears.

  5. Read the license text, select the I Accept the Terms of the License Agreement option, and click Next.
    The Installation Option page appears.

  6. Select New and click Next.
    The Installation Type page appears.
  7. Select Typical or Custom, based on the type of installation you want to perform and click Next.
    If you select Typical, the Installation Definition page appears. If you select Custom, the Components page appears.
  8. Select the components to install and click Next.
  9. Continue with the installation by entering the required information in each wizard page and clicking Next.
    After you complete the last data entry page of the wizard and click Next, the Review Settings page appears, listing the information that you entered.

  10. Review the information and, if it is correct, click Install.

    Note: To change an entry, click Back as many times as necessary to locate that entry. Then, make the appropriate change, click Next until the Review Settings page reappears, verify the change, and click Next again.

    The Monitor Progress page appears and the progress is displayed. When the installation completes, the Installation Complete page appears.

  11. Click Finish.
    The server installation is complete.

wa_setup.sh—Install, Update, or Remove CA Workload Automation AE

You can install, update, or remove CA Workload Automation AE using the shell script named wa_setup.sh from the CA Workload Automation AE DVD. It can also be used to generate a response file for a silent installation and to perform a silent installation. When you run wa_setup.sh, you specify the appropriate options depending on the functions you want to perform.

The following options are available:

  • -a response_file
    Creates only a response file with the specified name. You can use this response file during a later installation of CA Workload Automation AE.
  • -F
    Forces the CA Workload Automation AE installation, ignoring backup errors. When updating an existing installation, wa_setup.sh first backs up the existing installation in case the update fails. Normally, if the backup fails, the update cannot occur. However, this option specifies to ignore the backup errors and continue with the CA Workload Automation AE installation.
  • -h|-?
    Displays the usage statement.
  • -o
    Overlays the response file. Use this option with the -a option.

  • -p log_file
    Logs the installation in the specified file. If you do not specify this option, the installation log is stored in /opt/CA/installer/log.

    Note: If the installation abnormally terminates, the installation log is stored in /tmp or the directory specified in $TMPDIR.
  • -r response_file
    Installs CA Workload Automation AE using the specified response file. Specify the full path. Use this option with the -s option.
  • -s
    Installs CA Workload Automation AE in unattended (silent) mode. If you do not specify the -r option, default parameters are used.
  • -t trace_file
    Traces the installation to the specified file. Specify the full path.
  • -v
    Displays the version of the installer.
  • -x
    Extracts only CA Workload Automation AE to the /tmp directory or to the directory specified in $TMPDIR. The file name that is used for the extracted CA Workload Automation AE is internally defined and displayed on the command line.

  • -agent
    Installs only the agent. This option lets you bypass the installation wizard pages that are not directly associated with the typical agent installation.

    Note: You can also install the agent using the agent_setup.sh script. The agent that is installed using either wa_setup.sh or agent_setup.sh is the same.
  • -client
    Installs the client and agent. This option lets you bypass the installation wizard pages that are not directly associated with the typical client installation.
  • -server
    Installs the scheduler, application server, client, and agent. This option lets you bypass the installation wizard pages that are not directly associated with the typical server installation.
