Install CA WCC on UNIX Using Console Mode

This procedure describes how to install CA WCC on the CA WCC server using console mode.

Notes:

  • CA EEM must be running when you install CA WCC so that registration can occur; otherwise, you cannot install CA WCC.
  • Before you begin the installation, stop performance-monitoring, antivirus, and antispyware software.
  • If CA WCC is already installed on the server, you cannot reinstall CA WCC. Uninstall CA WCC first, then install CA WCC.
  • If the database you specify for the CA WCC database is used with another CA WCC instance, the option to install CA WCC in High Availability mode appears.

Follow these steps:

  1. Log in as root.

  2. Mount the CA WCC media.

    Note: We strongly recommend that you mount the ISO image instead of using an image "buster" tool. The path to the ISO image cannot contain space characters. The use of an image "buster" tool produces unexpected results.

  3. Change directories to the mounted file system and run the following command in a shell window:

    ./UnixInstaller.sh -console

    The installation option prompt appears.

  4. Enter 1 (Install CA Workload Control Center r11.4) and press Enter.
    The Introduction message appears.

  5. Press Enter.
    The end-user license agreement option panel appears or the installer prompts you to cancel the installation.

    Note: The installer prompts you to cancel only if CA WCC is already installed.

  6. Press Enter or type 2.

    Important! You can continue the installation only if you press Enter or type 2. If you type 3, the installation is cancelled. If you type 2, you accept the terms of the agreement without reading it. To read the agreement first, press Enter.

    The license agreement or prompt appears, depending on your selection.

  7. (As Applicable) Press Enter, read the text, and press Enter again. Repeat this action until you reach the end of the text, then type Y to accept the terms and press Enter.

    Important! The installation continues only if you accept the terms. To cancel the installation, type N and press Enter.

    The next prompt appears.

  8. Specify the installation directory path or accept the default path and press Enter.

    Note: Do not specify the root directory. For example, do not specify /.

    The next prompt appears.

  9. Continue with the installation, responding to each prompt as it appears.

    Note: The following rules apply:

    • For a prompt with multiple options, enter the option number you want (or follow the displayed instructions to accept the default selection indicated by the right arrow) and press Enter to complete the selection.

    • For a prompt that requires a single value, enter the correct value (or accept the default value enclosed in brackets) and press Enter.

    • If the value specified for a port is in use or is not in the defined range, you are prompted to specify a different value.

    • For a prompt for a password or for confirmation of a password, type your password and press Enter.

  10. Press Enter after you respond to the last prompt.

    The summary information appears, listing the components to install.

  11. Review the summary information and, if it is correct, press Enter.

    Note: To change to an entry, enter back as many times as necessary to locate that entry. Then, make the appropriate change, press Enter until the pre-installation summary appears again, and press Enter.

    The installation process begins, and the progress is displayed. When the installation completes, you are prompted to exit the installation.

  12. Press Enter.
    The CA WCC installation is complete.

  13. Enter 0.
    The installer closes.

    Notes:

    • Log out and log back in to the computer and verify that the CA WCC servers have started before you use or perform operations with CA WCC. You can display CA WCC server startup status by running the following command:

      grep "startup in" $CA_WCC_INSTALL_LOCATION/log/*.log

    • Installation log files are in the following location:

      WCC_installation_root/log/install
