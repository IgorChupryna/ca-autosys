(UNIX) Installation Scenarios

You must install the agent on every computer that you use to run jobs.

When you install the client, the agent is installed by default on the client computer.

Depending on your requirements, you can install additional agents on other computers using the wa_setup.sh script from the CA Workload Automation AE DVD or the agent_setup.sh script from the CA Workload Automation AE Standalone Agent DVD. We recommend the following guidelines:

  • Use the wa_setup.sh script to install the agent on any computer (including the server and client computers) if the computer is physically accessible. The wa_setup.sh script is large in size and may not be as efficient for distributing across your network.
  • Use the agent_setup.sh script to install the agent on remote computers or to install multiple agents on a single computer. The agent_setup.sh script is smaller is size than wa_setup.sh and can be distributed across the network.

Notes:

  • If you install the agent using the CA Workload Automation Agent DVD, the installer does not configure the agent to use all the available features of CA Workload Automation AE. We no longer recommend using this DVD to install the agent. If you used this method, we recommend that you take the following actions:
    1. Back up the agentparm.txt file.
    2. Uninstall the agent.
    3. Use the CA Workload Automation AE DVD or the CA Workload Automation AE Standalone Agent DVD to install the agent.
    4. Compare the backed up agentparm.txt file with the new agentparm.txt file and merge the appropriate parameters into the new agentparm.txt file.
  • If you are installing the agent for the first time, use the CA Workload Automation AE DVD or the CA Workload Automation AE Standalone Agent DVD. If you use this method, the agent is automatically configured to work with CA Workload Automation AE.
  • If you install only the agent and then the client or vice versa using the CA Workload Automation AE, CA Workload Automation AE Standalone Agent, or CA Workload Automation AE Standalone Client DVD, the instance_name.txt file is not created in the agent_install_directory/profiles directory and the EWAGLOBALPROFILE variable is not set and as a result the agent cannot use the /etc/auto.profile file to source the environment while running jobs. Perform the following steps for each agent so that it can use the /etc/auto.profile file to source the environment while running jobs:

    1. Create the instance_name.txt file in the agent_install_directory/profiles directory and add the following content:

      EWAGLOBALPROFILE=/etc/auto.profile

    2. Open the agentparm.txt file, add the following parameter, and save the file:

      oscomponent.environment.variable_manager_instance_name_SCH=agent_install_directory/profiles/instance_name.txt

    3. Restart the agent.

    Perform these steps if you installed multiple clients to access different CA Workload Automation AE instances. You do not need to perform these steps if you installed only one client for one CA Workload Automation AE instance and you do not need to read the /etc/auto.profile file.
