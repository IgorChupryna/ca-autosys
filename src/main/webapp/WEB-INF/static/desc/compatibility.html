Notes:

  • In this topic, CA Workload Automation AE refers to any or all components that can be installed using the CA Workload Automation AE DVD, CA Workload Automation AE Standalone DVDs (UNIX), or the CA Workload Automation Agent DVD.

  • To install these products on the same or separate computers, install CA EEM first, then CA Workload Automation AE, and then CA WCC. If these products are installed on the same computer, ensure that CA Workload Automation AE and CA WCC releases are at the same level (see table below). We recommend that you install CA EEM, CA Workload Automation AE components, CA WCC, and additional agents on separate computers as follows:
     

  • To upgrade these products installed on separate computers, upgrade CA EEM first, then CA WCC, and then CA Workload Automation AE.

  • To upgrade these products installed on the same computer, consider the following points

    • If the CA EEM server, CA Workload Automation AE server, and the CA WCC server are on the same computer, upgrade the CA EEM server first followed by the CA Workload Automation AE server and then the CA WCC server. The CA Workload Automation AE server must be upgraded before you restart the CA Workload Automation AE services. Ensure that you upgrade CA Workload Automation AE and CA WCC releases to the same level (see table below).

    • On HP, if SSA is installed using the CA Common Components DVD and the CA Workload Automation AE components are installed using any of the CA Workload Automation AE server or standalone DVDs on the same computer, upgrade CA Workload Automation AE first and then the common components using the CA Common Components DVD. If Command Sponsor Release 11.4 SP4 or previous is installed, upgrade CA Workload Automation AE first, then the common components using the CA Common Components DVD, and then the Command Sponsor. To upgrade the Command Sponsor, issue the following commands and follow the instructions on the installation wizard:

      cd CA Workload Automation AE_Install_Directory/CommandSponsor
      chmod +x WCC_CommandSponsor.bin
      ./WCC_CommandSponsor.bin

    • If the CA Workload Automation AE SDK and the CA WCC server are on the same computer, upgrade the CA WCC server and it will automatically upgrade the CA Workload Automation AE SDK to the same level.

    • If any of the CA Workload Automation AE components (except for the CA Workload Automation AE server) and the CA WCC server are on the same computer, upgrade CA Workload Automation AE first using the respective DVD (either CA Workload Automation AE Standalone Client or SDK or CA Workload Automation AE Standalone Agent DVD on UNIX or CA Workload Automation AE for Windows DVD on Windows) and then upgrade the CA WCC server.

  • Stop the CA WCC services before you upgrade CA Workload Automation AE. On Windows, you must also follow the steps in Installing CA Workload Automation AE and CA WCC on the Same Computer for a successful installation or upgrade.
