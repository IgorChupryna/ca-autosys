Support on Cloud

CA Workload Automation AE Release 11.3.6 and later and CA WCC Release 11.4 and later (including the agents) are supported on the cloud with the following database limitation:

  • Amazon RDS and Microsoft Azure PaaS databases are not supported.

Consider the following points when hosting CA Workload Automation AE or CA WCC on the cloud:

  • The CA Workload Automation AE database should be hosted within the same environment as the CA Workload Automation AE server for optimal performance. For example, hosting the database and the CA Workload Automation AE application servers on premise and the CA Workload Automation AE scheduler on the cloud will most likely significantly increase scheduler latency and general response times.
  • Enable the ports that are related to CA Workload Automation AE as well as traffic in the respective network.
  • In Amazon Web Services (AWS) environment, ensure that the IP address is resolvable across the respective domains to enable proper communication between the CA Technologies products. For example, you can resolve the IP address by updating the host file, providing the DNS server details, or by navigating to EC2 configService settings > EC2 Service properties and selecting the Set Computer Name option.

Agent Supported Versions

CA Workload Automation AE installs CA Workload Automation Agent for UNIX, Linux, or Windows 11.3.07 (Build 979). However, on HP-UX PA-RISC, CA Workload Automation AE installs CA Workload Automation Agent 11.3.05 (11.3 SP5) (Build 904). For more information on the agent or configuring the agent to behave like the legacy agent, see CA Workload Automation Agents.

Note: You cannot issue the autoping command with the -S option or run jobs that contain globs or blobs as input or output, if you installed:

  • The agent using the CA Workload Automation Agent for UNIX, Linux, or Windows DVD where the native CA Workload Automation AE client is not installed.
  • (HP-Itanium only) The agent using the CA Workload Automation AE DVD as the agent installed is 64-bit.
  • The 64-bit agent using the CA Workload Automation Agent for UNIX, Linux, or Windows DVD.

Patch 979306 -- Time Zone Update for Windows

For CA Workload Automation AE to work correctly with daylight saving time (DST), install patch 979306 on the CA Workload Automation AE client computers.

You can download the patch from http://support.microsoft.com/kb/955839.

For more information about configuring DST for Windows, see http://support.microsoft.com/kb/914387.

Note: The patch number is 979306 at the time of General Availability (GA). This patch may be superseded post GA. Ensure that you install the latest patch. For current information about the time zone patches, check the Microsoft Support web site.
